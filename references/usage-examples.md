# 使用示例

## 示例 1：初始化新小说项目

- **场景/输入**："我想写一个穿越玄幻小说" 或 `/nova init`
- **AI 行为**：问作品名+一句话定位 → 创建 `作品名/README.md` + `作品名/正文/` → `git init` → 询问是否关联 GitHub → commit
- **产出**：最小骨架，用户可以立刻开始写第一章

## 示例 2：口述设定并归档

- **场景/输入**："主角叫李凡，穿越者，在修仙世界觉醒了系统。系统可以兑换功法，但有副作用..."
- **AI 行为**：提取关键信息 → 生成/更新四个设定文件（作品定位、世界观、角色、大纲）
- **产出**：结构化的设定文件，后续写正文时自动参考

## 示例 3：批量修订

- **场景/输入**："把所有的门派改成学院，系统改成天赋觉醒"
- **AI 行为**：全局搜索"门派"和"系统" → 列出受影响文件 → 批量替换 → commit
- **产出**：所有相关文件中的术语统一

## 示例 4：续写

- **场景/输入**："继续上次写" 或 `/nova cont`
- **AI 行为**：读取正文 → 定位最后一个 `<!-- 断点 -->` 标记 → 回顾上下文 500 字 → 续写
- **产出**：从上次断点继续的正文

## 示例 5：文风分析

- **场景/输入**：用户粘贴一段 500-2000 字的小说文本 + "分析这段文风"
- **AI 行为**：拆解 7 个维度（句子长度、句式结构、段落组织、对话风格、描写密度、用词特征、节奏特征） → 写入 `参考文风.md` 和 `结构规则.md`
- **产出**：后续 `/nova write` 自动模仿此风格

## 示例 6：增加规则

- **场景/输入**："增加规则：不写悲剧结局"
- **AI 行为**：自动识别 → 写入 `内容规则.md` 的「剧情红线」章节 → commit
- **产出**：规则即时生效，后续生成内容遵循此约束

## 示例 7：查看进度

- **场景/输入**："进度" 或 `/nova stat`
- **AI 行为**：读取正文 → 统计当前章节位置 + 总字数 → 展示最近 5 条 commits → 检查未提交变更
- **产出**：一目了然的项目状态

## 示例 8：删除内容

- **场景/输入**："删了第三章" 或 "删除角色李凡"
- **AI 行为**：确认操作 → 列出受影响文件 → 用户再次确认 → 执行删除 → commit（可回滚）
- **产出**：安全删除，git 历史可恢复

## 注意事项

- 每次执行任务前读取 `.rules/` 下规则文件，优先遵守总规则和禁止词库
- 规则修改后自动 commit，同步写入规则变更日志
- 每次实质性修改都应 commit，push 仅在用户明确说"推送"时执行
