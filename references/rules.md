# 规则引擎

**AI 执行任务前必须读取 `.rules/` 下的规则文件。规则即法律，不可跳过。**

## 核心原则

1. **规则即法律**：每一条约束都是 AI 必须遵守的指令
2. **先读规则后执行**：每次对话开始时重新读取规则文件
3. **规则可随时修改**：修改后 AI 必须立即遵循
4. **规则纳入版本管理**：每次修改都生成 commit，可回溯
5. **高度可扩展**：用户可自定义规则文件，适配不同工作流需求

## 规则文件结构

```
.rules/
├── 规则引擎.md          # 必读：定义规则加载顺序、优先级、冲突处理
├── 禁止事项.md          # 必读：禁止的情节、桥段、词汇
├── 输出格式.md          # 必读：章节结构、对话标记
├── 文风规范.md          # 必读：句式、用词、描写密度
└── [自定义规则].md      # 可选：按需扩展，适配不同工作流
```

## 默认模板

AI 可辅助用户从模板创建规则文件：

| 模板文件 | 用途 |
|----------|------|
| `templates/规则引擎.md` | 规则引擎配置模板 |
| `templates/禁止事项.md` | 禁止事项模板（含内容/题材/词汇禁止） |
| `templates/输出格式.md` | 输出格式模板（含章节/对话/场景格式） |
| `templates/文风规范.md` | 文风规范模板（含句式/用词/节奏） |

## AI 辅助自定义规则引擎

**当用户要求"帮我定制规则"、"创建规则"、"修改规则引擎"时，AI 主动引导完成以下流程：**

### 流程一：理解需求

1. 询问用户想实现什么目标
2. 引导用户描述具体场景
3. AI 提炼出可规则化的要点

### 流程二：推荐规则方案

AI 根据用户需求，推荐规则结构：

| 需求类型 | 推荐规则文件 | 示例 |
|------|------|------|
| 禁止特定内容 | `.rules/禁止事项.md` | "禁止主角死亡" |
| 统一写作风格 | `.rules/文风规范.md` | "短句为主，不超过20字" |
| 控制输出结构 | `.rules/输出格式.md` | "对话用破折号开头" |
| 限制角色关系 | `.rules/人物关系.md` | "男二不得与女主单独相处" |
| 控制剧情节奏 | `.rules/剧情节奏.md` | "每章至少2个冲突点" |
| 统一专有词汇 | `.rules/术语表.md` | "系统→觉醒者" |

### 流程三：生成规则文件

1. AI 根据需求从模板生成规则初稿
2. 展示给用户确认
3. 用户修改/补充
4. AI 整合并写入 `.rules/` 目录
5. 执行 `git commit` 记录

### 流程四：测试验证

1. 用新规则生成一段测试文本
2. 展示给用户验证效果
3. 如需调整，返回流程三修改
4. 确认无误后正式生效

## Commit 规范

| 操作 | Commit 信息 |
|------|-------------|
| 新增禁止项 | `规则更新：禁止事项-xxx` |
| 修订规则 | `规则修订：文风规范-xxx` |
| 新增自定义规则 | `规则扩展：新增剧情节奏规则` |
| 删除规则 | `规则删除：xxx规则` |

```

```
