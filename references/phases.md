# 五阶段详细流程

## 阶段一：原始素材投喂

作者丢素材（文件或口述），AI 全量读取并理解。提取可结构化信息：角色、世界观、剧情走向、核心设定。

## 阶段二：结构化归档

整理为四份标准 Markdown 文件，存于 `作品名/` 目录。

| 文件 | 内容 |
|------|------|
| `作品名-作品定位.md` | 类型、尺度、感情线、CP、结局规划 |
| `作品名-世界观设定.md` | 世界观体系、规则、背景 |
| `作品名-角色设定.md` | 全部角色详细档案（表格+分类） |
| `作品名-剧情大纲.md` | 分章/分阶段剧情大纲 |

规则：原始素材保留不动→`作品名/原始素材/`，设定文件统一 Markdown，名称格式 `作品名-分类.md`，自动生成 README.md。

## 阶段三：Git 版本化

```
cd 作品名
git init && git add -A
git commit -m "作品名 世界书设定"
git branch -M main
gh repo create 用户名/仓库名 --private --source=. --remote=origin --push
```

### Commit 规范

| 前缀 | 用途 | 示例 |
|------|------|------|
| `设定更新：` | 新增设定或情节 | `设定更新：新增反派角色线` |
| `设定修订：` | 修改已有设定 | `设定修订：世界观术语统一` |
| `正文：` | 新章节或续写 | `正文：第一章·初遇` |
| `修改：` | 微调 | `修改：主角性格描述` |
| `整理：` | 结构调整 | `整理：合并重复设定` |

## 阶段四：迭代修订

流程：理解修改意图→搜索受影响文件→列出修改清单→批量修改→commit+push。每次一条 commit，随时回溯。

## 阶段五：搁置与续写

搁置：标注进度→commit+push。续写：git pull→读设定+大纲→确认断点→继续。

## 目录结构完整说明

```
项目根目录/作品名/
├── README.md
├── 作品名-作品定位.md
├── 作品名-世界观设定.md
├── 作品名-角色设定.md
├── 作品名-剧情大纲.md
├── .rules/
│   ├── 文风规范.md
│   ├── 输出格式.md
│   ├── 角色命名规则.md
│   └── 禁止事项.md
├── 原始素材/
├── 正文/
└── .git/
```

## .rules/ 规则系统

项目 `.rules/` 存放自定义规则。优先级：项目 `.rules/` > 全局默认 > 作者实时口述。模板见 `rules-templates.md`。
