# 工作流程

## 目录结构

```
作品名/
├── README.md、作品名-作品定位.md、作品名-世界观设定.md
├── 作品名-角色设定.md、作品名-剧情大纲.md
├── .rules/（文风规范.md、输出格式.md、角色命名规则.md、禁止事项.md）
├── 原始素材/、正文/、.git/
```

## 五阶段流程

### 阶段一：原始素材投喂
- **作者**：投放零散文件或口述
- **AI**：读取素材，理解设定、角色、世界观

### 阶段二：结构化归档
生成四份标准文件：

| 文件 | 内容 |
|------|------|
| 作品名-作品定位.md | 类型、尺度、感情线、CP、结局 |
| 作品名-世界观设定.md | 世界观体系、规则 |
| 作品名-角色设定.md | 角色档案（表格+分类） |
| 作品名-剧情大纲.md | 分章/分阶段剧情 |

### 阶段三：Git 版本化

```bash
git init && git add -A && git commit -m "作品名 世界书设定"
gh repo create 用户名/仓库名 --private --source=. --remote=origin --push
```

**Commit 规范**：

| 前缀 | 示例 |
|------|------|
| `设定更新：` | `设定更新：新增反派角色线` |
| `设定修订：` | `设定修订：世界观术语统一` |
| `正文：` | `正文：第一章·初遇` |
| `修改：` | `修改：主角性格描述` |
| `整理：` | `整理：合并重复设定` |

### 阶段四：迭代修订
1. 全局搜索受影响位置
2. 列出修改文件清单
3. 批量修改所有相关文件
4. `git commit` + `git push`

### 阶段五：搁置与续写
- **暂停时**：进度标注，commit 并 push
- **续写时**：读取设定和大纲，确认断点后继续

## 角色分工

| 作者 | AI |
|------|-----|
| 创意构思、设定决策、文风/内容创作 | 阅读素材、整理归档、Git管理、GitHub管理、全局修订、Commit编写 |
