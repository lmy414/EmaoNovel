# 工作流程

## 目录结构（渐进生长）

```
init 后：                    feed 后：                      rule + write 后：
作品名/                      作品名/                        作品名/
├── README.md               ├── README.md                  ├── README.md
└── 正文/                    ├── 作品名-作品定位.md           ├── 作品名-作品定位.md
                             ├── 作品名-世界观设定.md          ├── 作品名-世界观设定.md
                             ├── 作品名-角色设定.md            ├── 作品名-角色设定.md
                             ├── 作品名-剧情大纲.md            ├── 作品名-剧情大纲.md
                             ├── 原始素材/                    ├── .rules/
                             ├── 正文/                       │   ├── 文风规范.md
                             └── .git/                       │   ├── 输出格式.md
                                                             │   └── 禁止事项.md
                                                             ├── 原始素材/
                                                             ├── 正文/
                                                             │   └── 第一章.md
                                                             └── .git/
```

项目不是一次性建完的——init 只给最小骨架，后续按需生长。

## 操作指南

### 初始化（`/nova init`）

1. 问作品名 + 一句话定位（如"穿越修仙+学院流，清水向"）
2. 创建 `作品名/README.md`（写作品名+定位）
3. 创建 `作品名/正文/` 空目录
4. `git init` → 询问是否关联 GitHub 远端 → commit

### 素材整理（`/nova feed`）

**首次 feed**：从零散素材中提取信息，生成四类设定文件。
**后续 feed**：读取已有设定文件，将新素材合并进去。

| 文件 | 内容 |
|------|------|
| 作品名-作品定位.md | 类型、尺度、感情线、CP、结局 |
| 作品名-世界观设定.md | 世界观体系、规则 |
| 作品名-角色设定.md | 角色档案（表格+分类） |
| 作品名-剧情大纲.md | 分章/分阶段剧情 |

### Git 版本管理

```bash
git init && git add -A && git commit -m "作品名：项目初始化"
```

**Commit 规范**：

| 前缀 | 示例 |
|------|------|
| `设定更新：` | `设定更新：新增反派角色线` |
| `设定修订：` | `设定修订：世界观术语统一` |
| `正文：` | `正文：第一章·初遇` |
| `修改：` | `修改：主角性格描述` |
| `整理：` | `整理：合并重复设定` |
| `规则：` | `规则：新增禁止事项` |

### 迭代修订（`/nova rev`）

1. 全局搜索受影响位置
2. 列出修改文件清单
3. 批刾修改所有相关文件
4. `git commit`
5. 用户说推送时再 `git push`

### 篇幅管理

长篇作品建议按"卷"或"篇章"拆分，避免单个文件过大：

```
正文/
├── 第一卷/
│   ├── 第一章.md
│   ├── 第二章.md
│   └── ...
└── 第二卷/
    ├── 第一章.md
    └── ...
```

### 搁置与续写

- **暂停时**（`/nova done`）：在当前文件末尾标注 `<!-- 断点：第X章第Y节 -->`，commit，用户说推送时 push
- **续写时**（`/nova cont`）：读取 README + 已有设定文件，定位断点，回顾上下文后继续

## 角色分工

| 作者 | AI |
|------|-----|
| 创意构思、设定决策、文风/内容创作 | 阅读素材、整理归档、Git 管理、GitHub 管理、全局修订、Commit 编写 |
