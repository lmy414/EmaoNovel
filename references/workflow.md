# 工作流程

## 执行入口

### 第一步：检查项目文件夹

1. 检查当前目录是否存在项目文件夹（如 `作品名/` 或 `novel/`）
2. 检查是否存在 `README.md` 作为项目标识

### 第二步：分支处理

**新项目（不存在）：**
1. 询问作品名称 + 一句话定位（如"穿越修仙 + 学院流"）
2. 创建最小骨架：
   ```
   作品名/
   ├── README.md      # 作品名 + 一句话定位
   └── 正文/           # 空目录，等待第一章
   ```
3. `git init` → 询问是否关联 GitHub 远端仓库 → 首次 commit

**已有项目（存在）：**
1. 读取 `README.md` 了解项目状态
2. 读取已存在的设定文件（如有）
3. 读取 `.rules/` 下规则文件（如有）
4. 确认后执行用户命令

### 第三步：执行用户命令

按用户意图执行；项目结构按需生长：

| 命令 | 自然语言 | AI 行为 | 产出 |
|------|----------|----------|------|
| `/nova init` | 开个新小说 / 初始化项目 | 问作品名+一句话定位 → 创建最小骨架 → git init | `README.md` + `正文/` |
| `/nova feed` | 帮我整理这些设定 | 读取原始素材 → 提取信息 → 归类写入设定文件 | `作品定位.md`、`世界观设定.md`、`角色设定.md`、`剧情大纲.md`、`原始素材/` |
| `/nova write` | 写第X章 / 继续写 | 读取大纲 + 规则 → 按文风规范生成正文 | `正文/第X章.md` |
| `/nova cont` | 继续上次写 | 读取 `<!-- 断点 -->` 标记 → 回顾上下文 → 续写 | 正文续写 |
| `/nova done` | 今天先到这 | 写入 `<!-- 断点：第X章第Y节 -->` → commit → push | 进度归档 |
| `/nova rev` | 把XX改成YY / 全局替换 | 全局搜索受影响位置 → 列出清单 → 批量修改 → commit | commit 记录变更 |
| `/nova delete` | 删了XXX / 删除XXX | 确认 → 列出受影响文件 → 再次确认 → 执行删除 → 清理关联引用 → commit | git 历史可回滚 |
| `/nova rule` | 加条规则 / 改禁止事项 | 引导创建或修改 `.rules/` 规则文件 → commit | `.rules/` 下规则文件按需生长 |
| "增加规则：XXX" | — | 自动识别规则类型 → 写入对应规则文件的对应章节 → commit | 例如："不写悲剧结局" → `内容规则.md` 剧情红线章节 |
| `/nova style` | 分析这段文风 | 拆解句式/用词/节奏 → 写入 `参考文风.md` + 更新 `结构规则.md` | 后续 `/nova write` 自动模仿 |
| `/nova stat` | 进度 / 写到哪了 | 查看项目状态 | 当前章节 + 总字数 + 最近 5 commits + 未提交变更 |

                             └── .git/                       │   ├── 版本管理规则.md
                                                             │   ├── 结构规则.md

项目不是一次性建完的——init 只给最小骨架，后续按需生长。

## 操作细节

以下为命令映射表中未覆盖的细节。

### 素材整理（`/nova feed`）

**首次 feed**：从零散素材中提取信息，生成四类设定文件。
**后续 feed**：读取已有设定文件，将新素材合并进去。

| 文件 | 内容 |
|------|------|
| 作品名-作品定位.md | 类型、尺度、感情线、CP、结局 |
| 作品名-世界观设定.md | 世界观体系、规则 |
| 作品名-角色设定.md | 角色档案（表格+分类） |
| 作品名-剧情大纲.md | 分章/分阶段剧情 |

### Git 版本管理

```bash
git init && git add -A && git commit -m "作品名：项目初始化"
```

Commit 规范见 [.rules/版本管理规则.md](templates/版本管理规则.md)（模板），各项目以 `.rules/` 下的副本为准。

### 规则管理

当用户说"增加规则：XXX"或"加条规则：XXX"，AI 按以下三步处理，无需用户指定放到哪里：

**第一步：识别内容 → 匹配落点**

| 你说... | AI 自动写入... |
|---------|---------------|
| "增加规则：禁用'卧槽'" | `禁止词库.md` → 自定义禁止词章节 |
| "增加规则：短句为主，不超过 20 字" | `结构规则.md` → 文风规范章节 |
| "增加规则：对话用破折号" | `结构规则.md` → 输出格式章节 |
| "增加规则：男二不得与女主单独相处" | `内容规则.md` → 角色关系约束章节 |
| "增加规则：不写悲剧结局" | `内容规则.md` → 剧情红线章节 |
| "增加规则：这个世界没有复活术" | `内容规则.md` → 世界观一致性章节 |
| "增加规则：灵力不能写成法力" | `内容规则.md` → 术语统一章节 |
| "增加规则：战斗升级体系" | `自定义规则.md` → 索引新文件 `战斗体系规则.md` |

**第二步：直接写入** — AI 将规则写入对应文件的对应章节，无需逐条确认。写入后简短告知用户写到哪个文件即可。

**第三步：commit** — 格式见 `.rules/版本管理规则.md`。

修改或删除已有规则时，AI 全局搜索规则文件，找到匹配内容后执行。多条规则批量操作时，逐条识别 → 逐条写入 → 合并为一个 commit。

### 文风分析

当用户粘贴文本并说"分析这段文风"/"模仿这个风格"/"按这个风格写"时：

**第一步：拆解 7 个维度**

| 维度 | 提取内容 |
|------|---------|
| 句子长度 | 平均字数 → 短句(≤15字) / 中句(15-30字) / 长句(≥30字) / 长短交替 |
| 句式结构 | 主谓宾完整句 / 省略主语 / 口语化 / 书面化 |
| 段落组织 | 平均段长、分段频率、是否用单句成段制造节奏 |
| 对话风格 | 引号格式、语气特征、对话与叙述的比例 |
| 描写密度 | 动作/心理/环境描写的比例和详略程度 |
| 用词特征 | 白话/文白夹杂/华丽/朴素、高频句式结构 |
| 节奏特征 | 场景切换频率、高潮铺垫方式 |

**第二步：写入** — 将原始文本 + 拆解分析写入 `参考文风.md`，同时将拆解结果转化为规则参数更新 `结构规则.md` 的文风规范章节。

**第三步：commit** — `git commit -m "规则：文风分析写入"`。

用户确认后，后续 `/nova write` 自动按此风格生成正文。

### 篇幅管理

长篇作品建议按"卷"或"篇章"拆分，避免单个文件过大：

```
正文/
├── 第一卷/
│   ├── 第一章.md
│   ├── 第二章.md
│   └── ...
└── 第二卷/
    ├── 第一章.md
    └── ...
```

### 搁置与续写

- **暂停时**（`/nova done`）：在当前文件末尾标注 `<!-- 断点：第X章第Y节 -->`，commit，用户说推送时 push
- **续写时**（`/nova cont`）：读取 README + 已有设定文件，定位断点，回顾上下文后继续
