# 检查清单

**AI 在关键节点必须逐项自检，不跳过。**

---

## 写正文后（`/nova write`）

```
□ 对话格式：是否符合结构规则中定义的格式（引号/破折号）？
□ 段落长度：是否有超过 200 字的段落？
□ 禁止词库：是否出现禁止词库中的任何词汇？
□ 敏感词库：是否出现敏感词？如有，提醒用户
□ 术语统一：世界观术语是否一致（如"灵力"未写成"法力"）？
□ 内容红线：是否违反内容规则中的角色关系/剧情红线？
□ 文风一致：句式长度、描写密度是否符合结构规则的文风要求？
□ 心理活动：如有心理描写，是否使用 [*] 标记？
```

---

## 修订后（`/nova rev`）

```
□ 搜索完整性：是否搜索了全部受影响文件（设定文件 + 正文 + 大纲）？
□ 术语一致性：替换后是否出现术语不统一？（如改了"门派"但漏了"宗门"）
□ 连带影响：修改是否影响其他设定？（如删了角色，是否清理了该角色的关系线）
□ commit：修改完成后是否 commit？
```

---

## 整理设定后（`/nova feed`）

```
□ 设定冲突：新素材是否与已有设定矛盾？（如已有"九阶灵力"，新素材说"十阶"）
□ 重复内容：是否已有同名角色/同功能设定？
□ 文件齐全：四类设定文件是否都已更新？
□ commit：归档完成后是否 commit？
```

---

## 存档前（`/nova done`）

```
□ 进度标注：是否在正文末尾标了断点 `<!-- 断点：第X章第Y节 -->`？
□ 未提交变更：`git status` 是否有未 commit 的文件？
□ 规则变更：本次会话是否修改了规则？如有，是否写入规则变更日志？
□ push：用户是否明确说了"推送"？
```
