# 结构规则

---

## 项目目录结构

```
作品名/
├── README.md
├── 作品名-作品定位.md
├── 作品名-世界观设定.md
├── 作品名-角色设定.md
├── 作品名-剧情大纲.md
├── .rules/                  # 本规则目录
├── 原始素材/                 # 用户投喂的零散素材
├── 正文/                     # 章节文件
└── .git/
```

---

## 文件命名规范

- 设定文件：`作品名-设定类型.md`（如 `修仙传-角色设定.md`）
- 章节文件：`正文/第X章.md` 或 `正文/第X卷/第Y章.md`
- 规则文件：`.rules/规则名.md`

---

## 输出格式

### 章节结构
```
# 第X章 章节标题

[正文段落...]

```

### 段落格式
- 段落之间空一行
- 每段不超过 200 字
- 章节内场景切换用 `---` 分隔

### 对话格式
- 对话用 `"内容。"角色说。` 格式
- 或破折号格式：`——对话内容。`

### 特殊标记
- 心理活动：`[*]他心想，这次一定要成功。`
- 旁白：无标记，直接叙述

---

## 断点规范

暂停写作时，AI 在正文末尾写入断点标记：

```
<!-- 断点：第X章第Y节 | 日期 -->
```

续写时，AI 读取正文文件，定位最后一个 `<!-- 断点` 标记，回顾标记前的 500 字上下文后继续。

---

## 文风规范

| 维度 | 默认设置 |
|------|----------|
| 句子长度 | 短句为主，不超过 20 字 |
| 段落长度 | 每段不超过 200 字 |
| 对话语气 | 口语化 |
| 每句对话 | 不超过 50 字 |
| 描写密度 | 动作描写简洁，心理描写适度，环境描写按需 |
| 章节节奏 | 张弛有度 |

如需参考特定文风，将范文放入 [参考文风.md](参考文风.md)，AI 在生成正文时加载。

## 人机分工

| 作者 | AI |
|------|-----|
| 创意构思、设定决策 | 读取素材、理解脉络 |
| 决定剧情走向 | 整理为结构化设定文件 |
| 写正文、定文风 | 按规则生成正文草稿 |
| 说出修改指令 | 全局搜索 → 批量修订 → commit |
| 说"推送"或"今天先到这" | commit + push |
| 做创作决策 | 做文件整理、版本管理、格式统一 |
