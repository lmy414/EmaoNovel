# 版本管理规则

---

## Git 初始化

```
git init
git remote add origin <仓库地址>   # 用户确认后执行
```

---

## Commit 规范

| 前缀 | 用途 | 示例 |
|------|------|------|
| `设定：` | 新增设定内容 | `设定：新增反派角色线` |
| `修订：` | 修改已有内容 | `修订：世界观术语统一` |
| `正文：` | 正文创作 | `正文：第一章·初遇` |
| `规则：` | 规则增删改 | `规则：新增禁止词汇` |
| `文风：` | 文风分析/修改 | `文风：分析写入参考风格` |
| `删除：` | 删除内容 | `删除：第三章` |
| `整理：` | 文件重组织 | `整理：合并重复设定文件` |
| `进度：` | 存档/断点 | `进度：第三章第二节断点` |

每次实质性修改都应单独 commit，commit message 格式：`前缀 简短说明`。

---

## Push 时机

- commit 自动执行
- push 在用户明确说"推送"、"push"、"今天先到这"时执行
- 不自动 push

---

## 分支策略

- 默认在 `main` 分支工作
- 大范围修订或实验性修改时，建议切分支：`git checkout -b 修订-<内容>`
- 完成后合并回 `main`

---

## 删除操作

破坏性操作，必须两步确认：

1. 用户说"删了XXX" → AI 搜索受影响文件 → 列出清单 → 问"确认删除以下文件？"
2. 用户确认 → 执行删除 → 清理其他文件中的关联引用 → commit: `删除：XXX`
3. 撤销方式：`git checkout <commit> -- <文件路径>`

## 回滚

- 查看历史：`git log --oneline`
- 回退单个文件：`git checkout <commit> -- <文件路径>`
- 用户确认后执行
