# 小说创作工作流

作者负责"写什么"，AI 负责"怎么存"。详细流程见 `references/phases.md`。

## 执行流程

```
用户触发（"帮我整理***文档设定"）
        │
        ▼
检查工作目录下是否有对应作品文件夹
        │
   ┌────┴────┐
  有          无
   │           │
   ▼           ▼
优先读取    初始化完整目录结构：
.rules/     作品名/
全部文件     ├── README.md
（文风/输出/ ├── 作品名-作品定位.md
命名/禁忌）  ├── 作品名-世界观设定.md
理解约束后  ├── 作品名-角色设定.md
进入对应    ├── 作品名-剧情大纲.md
阶段        ├── .rules/
            │   ├── 文风规范.md
            │   ├── 输出格式.md
            │   ├── 角色命名规则.md
            │   └── 禁止事项.md
            ├── 原始素材/
            ├── 正文/
            └── .git/（待git init）
            │
            ▼
            阶段一：读取素材 → 归档

后续执行依据 .rules/ + 用户指令。规则贯穿全流程。
```

## 命令参考

| 作者说 | AI 做 |
|---|---|
| "帮我整理这些设定" 丢素材 | 阶段一→二：读取→归档为四份标准文件 |
| "初始化小说项目" | 阶段三：git init → 创建私有仓库 → 推送 |
| "把XX改成YY" "删掉那部分" | 阶段四：全局搜索→批量修改→commit+push |
| "继续写" "上次写到哪了" | 阶段五：git pull→读设定→确认断点→续写 |
| "先存档" "今天先到这" | 标注进度→commit+push |
| "改下文风规范" | 修改 .rules/ 文件→commit+push |

## Commit 规范

| 前缀 | 用途 |
|---|---|
| `设定更新：` | 新增设定或情节 |
| `设定修订：` | 修改已有设定 |
| `正文：` | 新章节或续写 |
| `修改：` | 微调 |
| `整理：` | 结构调整 |

.rules/ 模板见 `references/rules-templates.md`。完整流程见 `references/phases.md`。
