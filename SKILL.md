# 基于 Git 的小说创作工作流

## 概述

以 Git 为版本管理核心、AI 助手为执行层的小说创作工作流。作者负责"写什么"，AI 负责"怎么存"。

## 任务目标

- 本 Skill 用于：小说创作项目的高效管理与迭代
- 能力包含：原始素材归档、四类设定文件生成、Git 版本管理、批量修订、断点续写
- 触发条件：开始新项目、口述设定、整理角色、修订正文

## 执行入口流程

### 第一步：检查项目文件夹

1. 检查当前目录是否存在项目文件夹（如 `作品名/` 或 `novel/`）
2. 检查是否存在 `README.md` 作为项目标识

### 第二步：分支处理

**新项目（不存在）：**
1. 询问作品名称 + 一句话定位（如"穿越修仙 + 学院流"）
2. 创建最小骨架：
   ```
   作品名/
   ├── README.md      # 作品名 + 一句话定位
   └── 正文/           # 空目录，等待第一章
   ```
3. `git init` → 询问是否关联 GitHub 远端仓库 → 首次 commit

**已有项目（存在）：**
1. 读取 `README.md` 了解项目状态
2. 读取已存在的设定文件（如有）
3. 读取 `.rules/` 下规则文件（如有）
4. 确认后执行用户命令

### 第三步：执行用户命令

按用户意图执行；项目结构按需生长：

| 用户说... | AI 做什么 | 长出什么 |
|-----------|----------|----------|
| `/nova feed` 或 "帮我整理设定" | 从素材中提取信息 | `作品名-作品定位.md`、`世界观设定.md`、`角色设定.md`、`剧情大纲.md`、`原始素材/` |
| `/nova write` 或 "写第一章" | 读取规则 + 大纲，生成正文 | `正文/第一章.md` |
| `/nova rule` 或 "加条规则" | 从模板创建规则 | `.rules/文风规范.md`、`输出格式.md`、`禁止事项.md` |
| `/nova rev` 或 "把XX改成YY" | 全局搜索 → 批量替换 | commit 记录变更 |
| `/nova cont` 或 "继续上次写" | 读断点 → 续写 | 正文续写 |
| `/nova done` 或 "今天先到这" | 标断点 → commit → push | 进度归档 |

## 命令参考

自然语言触发为主，以下命令用于需要精确控制时：

| 命令 | 等价自然语言 | 做什么 |
|------|-------------|--------|
| `/nova init` | 开个新小说 / 初始化项目 | 问作品名+一句话定位 → 创建最小骨架 → git init |
| `/nova feed` | 帮我整理这些设定 | 读取原始素材 → 提取信息 → 归类写入设定文件 |
| `/nova rev` | 把XX改成YY / 全局替换 | 全局搜索受影响位置 → 列出清单 → 批量修改 → commit |
| `/nova write` | 写第X章 / 继续写 | 读取大纲 + 规则 → 按文风规范生成正文 |
| `/nova cont` | 继续上次写 | 读取断点标记 → 回顾上下文 → 续写 |
| `/nova done` | 今天先到这 | 标注进度断点 → commit → push |
| `/nova stat` | 进度 / 写到哪了 | 显示当前章节位置 + 最近 commits + 变更摘要 |
| `/nova rule` | 加条规则 / 改禁止事项 | 引导创建或修改 `.rules/` 规则文件 → commit |
